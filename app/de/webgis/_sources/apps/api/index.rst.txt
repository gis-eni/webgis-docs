WebGIS API
==========

Die **WebGIS API** ist das Herzstück der WebGIS-Plattform und bietet leistungsstarke Schnittstellen für den Zugriff auf und die Interaktion mit Kartendiensten. Sie stellt eine **REST API** zur Verfügung, über die Nutzer Karten- und Geodaten anfragen können. Darüber hinaus wird auch eine **Javascript API** angeboten, die die REST API-Aufrufe kapselt und so eine vereinfachte Integration in browserbasierte Anwendungen ermöglicht.

======================

Hauptfunktionen der WebGIS API:
-------------------------------
- **REST API** für den Abruf von Kartendiensten und Geodaten.
- **Javascript API** für die einfache Entwicklung von browserbasierten Anwendungen.
- Bereitstellung von Kartendiensten basierend auf Konfigurationen aus dem **WebGIS CMS**.
- Unterstützung von Abfragen und Visualisierungen von Geodaten.
- Flexibilität für Entwickler, eigene Kartenanwendungen und Services zu erstellen.

Über die **WebGIS API** können Kartenanwendungen mit verschiedenen Funktionen wie Layern, Abfragen und geografischen Messungen realisiert werden. Sie ist die zentrale Schnittstelle für Entwickler, die maßgeschneiderte WebGIS-Anwendungen bauen wollen.

======================

Verwendung der WebGIS API:
---------------------------
- **Entwickler**, die mit Geodaten arbeiten und eigene Webanwendungen aufbauen möchten, verwenden die **REST API** oder **Javascript API** verwenden.
- **Administrator:innen** können mit der API Kartendienste und deren Parameter über das **WebGIS CMS** definieren und anpassen.
- Durch die XML-Dateien, die durch das CMS generiert werden, können verschiedene Kartendienste und Themen schnell konfiguriert und über die API bereitgestellt werden.

Weitere Details zur **Javascript API** finden sich in der umfassenden Dokumentation unter (https://docs.webgiscloud.com/dev/index.html).
